Complete Guide to Deploying a MERN Stack Application on Netlify

Follow this comprehensive guide to deploy your MERN stack application with the frontend hosted on Netlify and the backend on a different hosting service like Heroku.

  1. Create a Netlify Account:

    Visit Netlify's website and sign up for an account. Verify your email and set up your profile.

  2. Prepare Your Application:

    Ensure your MERN stack application is production-ready:

    • Frontend: Your React app should have a build script in package.json and be ready for deployment.
    • Backend: Ensure your Node.js server is configured properly and deployed separately on another service like Heroku.
  3. Push Your Frontend Code to GitHub:

    Push your frontend React code to a GitHub repository. Netlify integrates with GitHub for deployments.

  4. Deploy Frontend on Netlify:

    Set up and deploy your frontend application using Netlify:

    1. Connect GitHub Repository:

      In Netlify, click on New site from Git, connect your GitHub account, and select the repository with your React app.

    2. Configure Build Settings:

      Set the build command to npm run build and the publish directory to build.

      
Build command: npm run build
Publish directory: build
    3. Deploy Your Site:

      Click Deploy site and Netlify will build and deploy your React application. Monitor the process and view logs in the Netlify dashboard.

  5. Deploy Backend on Heroku (or Another Service):

    Ensure your Node.js backend is deployed on a different hosting service. This guide assumes Heroku, but you can use any service you prefer.

    1. Prepare Your Node.js App:

      Ensure your Node.js application has a package.json with a start script and any necessary configurations.

    2. Deploy to Heroku:

      Deploy your code to Heroku using the heroku CLI:

      
heroku create
git push heroku main
    3. Configure Environment Variables:

      Set up any necessary environment variables on Heroku, such as database connection strings and API keys:

      
heroku config:set MONGO_URI=your_mongo_db_uri
heroku config:set PORT=your_app_port
  6. Connect Frontend to Backend:

    Update your frontend code to use the backend API URL provided by Heroku (or your chosen backend service). You can set this URL in environment variables or directly in your frontend code.

  7. Verify and Test Your Application:

    After deployment, verify both frontend and backend by visiting their respective URLs. Test all features thoroughly to ensure everything is working correctly.

Frontend Deployment Details

  1. Prepare Your React Application:

    Ensure your React app is production-ready:

    • Build Configuration: Confirm that package.json has a build script: "build": "react-scripts build".
    • Environment Variables: Configure any necessary environment variables in .env files.
  2. Build Your React Application:

    Run npm run build to generate the production-ready build in the build directory.

  3. Connect to GitHub:

    Push your code to a GitHub repository if you haven’t already.

  4. Deploy on Netlify:

    In Netlify, connect your GitHub repository and configure the build settings as described above.

Backend Deployment Details

  1. Prepare Your Node.js Application:

    Ensure your Node.js backend is production-ready:

    • Build Configuration: Make sure package.json has a start script: "start": "node server.js".
    • Environment Variables: Configure environment variables in your hosting service.
  2. Deploy to Heroku:

    Use the Heroku CLI to deploy your backend application.

  3. Configure Environment Variables:

    Set up necessary environment variables on Heroku as needed.

Common Errors and Fixes

Missing Environment Variables

Issue: The application may not function correctly if required environment variables are missing.

Fix: Add necessary environment variables in Netlify’s environment settings and Heroku’s config vars.

Build Failures

Issue: Build process may fail due to incorrect build commands or missing dependencies.

Fix: Verify build scripts in package.json and check logs for errors.

Deployment Errors

Issue: Errors during deployment can occur due to various configuration issues.

Fix: Review deployment logs on Netlify and Heroku for detailed error messages and adjust configurations as needed.

CORS Issues

Issue: Frontend might face issues with backend requests if CORS isn’t configured.

Fix: Set up CORS in your backend server. For Express:


const cors = require('cors');
app.use(cors());
