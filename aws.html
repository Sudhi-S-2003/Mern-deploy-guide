Complete Guide to Deploying a MERN Stack Application on AWS

Follow this comprehensive guide to deploy your MERN stack application on AWS using Elastic Beanstalk and S3 with CloudFront.

  1. Create an AWS Account:

    Visit AWS's website and sign up for an account. Verify your email and set up billing.

  2. Prepare Your Application:

    Ensure your MERN stack application is production-ready:

    • Frontend: Your React app should be build-ready with a build script in package.json.
    • Backend: Your Node.js server should be ready to handle requests and serve static files if needed.
  3. Push Your Code to GitHub:

    Push your code to a GitHub repository. AWS integrates with GitHub for deployments.

  4. Deploy Frontend to S3 and CloudFront:

    Set up an S3 bucket to host your React build files and use CloudFront for CDN. Here’s how:

    1. Create an S3 Bucket:

      Log in to AWS Management Console, navigate to S3, and create a new bucket. Ensure the bucket is set for public access if it will be serving static files.

    2. Upload Build Files:

      Build your React app locally with npm run build and upload the contents of the build directory to your S3 bucket.

    3. Configure CloudFront:

      Set up CloudFront to serve your content from the S3 bucket. Create a new CloudFront distribution, set the origin to your S3 bucket, and configure caching and access settings.

  5. Deploy Backend to Elastic Beanstalk:

    Deploy your Node.js backend application using Elastic Beanstalk:

    1. Create an Elastic Beanstalk Application:

      In the AWS Management Console, navigate to Elastic Beanstalk, and create a new application. Choose the Node.js platform and configure the environment settings.

    2. Deploy Your Code:

      Upload your Node.js application code (ZIP file) or connect your GitHub repository. Elastic Beanstalk will handle deployment and provisioning of resources.

    3. Configure Environment Variables:

      Add necessary environment variables in the Elastic Beanstalk configuration. These might include database connection strings and API keys.

  6. Set Up Domain Name (Optional):

    Configure a custom domain name using Route 53 or another DNS service. Point your domain to the CloudFront distribution for the frontend and the Elastic Beanstalk environment for the backend.

  7. Verify and Test Your Application:

    After deployment, verify the frontend and backend by visiting their respective URLs. Test all features and functionality to ensure everything works as expected.

Frontend Deployment Details

  1. Prepare Your React Application:

    Ensure your React app is production-ready:

    • Build Configuration: Confirm that package.json has a build script: "build": "react-scripts build".
    • Environment Variables: Configure any necessary environment variables in .env files.
  2. Build Your React Application:

    Run npm run build to generate the production-ready build in the build directory.

  3. Upload to S3:

    Upload the contents of the build directory to your S3 bucket.

  4. Configure CloudFront:

    Set up CloudFront to serve your static files from S3. Configure caching and security settings as needed.

Backend Deployment Details

  1. Prepare Your Node.js Application:

    Ensure your Node.js backend is ready for deployment:

    • Build Configuration: Make sure package.json has a start script: "start": "node server.js".
    • Environment Variables: Set up environment variables for your application in Elastic Beanstalk.
  2. Deploy to Elastic Beanstalk:

    Upload your backend code or connect to your GitHub repository, and follow the prompts to deploy your application.

  3. Configure Environment:

    Add necessary environment variables and configuration settings in the Elastic Beanstalk environment settings.

Common Errors and Fixes

Missing Environment Variables

Issue: The application may not function correctly if required environment variables are missing.

Fix: Add necessary environment variables in the Elastic Beanstalk configuration and ensure they match those used in your application.

Build Failures

Issue: Build process may fail due to incorrect build commands or missing dependencies.

Fix: Verify build scripts in package.json and ensure all dependencies are correctly specified.

S3 Bucket Permissions

Issue: Incorrect S3 bucket permissions may prevent access to static files.

Fix: Ensure the S3 bucket has proper permissions and public access settings, if required.

CloudFront Caching Issues

Issue: CloudFront might cache old content or misconfigure distribution settings.

Fix: Clear the CloudFront cache and review distribution settings for correct configuration.

Elastic Beanstalk Configuration Errors

Issue: Misconfiguration of Elastic Beanstalk environment settings or missing application files.

Fix: Review Elastic Beanstalk logs and configuration settings. Ensure all application files are correctly uploaded.

CORS Issues

Issue: Frontend might face issues with backend requests if CORS isn’t configured.

Fix: Set up CORS in your backend server. For Express:


const cors = require('cors');
app.use(cors());
